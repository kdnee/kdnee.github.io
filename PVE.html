Server Management

One of my favorite projects so far has definitely been building and setting up my Proxmox server. Proxmox uses the KVM hypervisor and LXC(Linux Containers) with a web-based UI to manage it all. I started the build with an MSI FM2-A85XMA-E35 micro-ATX board with an AMD A10-5800K APU with 8 GB of memory and two hard drives for storage. Basically, it was my first pc all over again but instead of booting and installing windows, I would be installing the open-source server management platform, Proxmox. After hitting running into many different errors throughout the booting process, I decided to try running Proxmox on top of an install of Debian Buster instead of on the “bare-metal” system. After installing Proxmox through the repository and following all of the steps on the Proxmox Wiki, it was finally up.

I had access to the server via my PC and started to create a virtual machine to test out all of the features. This is where I ran into a rather critical error, the virtual machine had no internet connection. Checking all of the network settings in the Debian machine, the Proxmox instance, and the virtual machine and everything seemed fine. I decided to reboot the whole system, and that was definitely a bad idea. Not only could I no longer open the web UI, but even in Debian, it would not start up properly. After a few days of trying to find out what I had done wrong by searching for the issues of forums and wikis, I looked at something I should have looked at well before I started. The minimum system requirements, rookie move for sure. The motherboard I was using had no designated virtualization support (AMD-V) built into the bios or board, so even though the APU was capable of virtualization Proxmox didn’t respond well to splitting the network adapter because of the lack of virtualization of the board.

Thankfully I had a stash of old computer parts to pick from and was able to rebuild the server with an Intel Core i3-3220. Which is a year newer and effectively 9% faster according to UserBenchmark. With the proper hardware finally installed I was able to get a stable running version of Proxmox, I have even started using it to host my DNS sinkhole server Pi-Hole.
